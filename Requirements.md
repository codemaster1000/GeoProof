Project NameGeoProof - GPS Camera & Field Documentation App 1. OverviewGeoProof is an Android-first GPS Camera application designed for field workers, contractors, property inspectors, surveyors, delivery personnel, and other professionals who need verifiable photographic documentation. The app allows users to capture photos with embedded GPS coordinates, timestamps, project information, and custom notes, and generate professional PDF reports. The app operates entirely offline and requires no user account for basic functionality. The engineering stack is built on React Native and the Expo Managed Workflow to maintain a zero-local-build footprint. 2. Problem StatementProfessionals working in the field need clear evidence that work was completed at a specific location and time. Current market vulnerabilities (from Timemark, Timeprint, and Conota):Competitors force aggressive recurring subscriptions (up to ₹230/month or ₹1,050/year) just to use standard business utilities. Intrusive, unpredictable video ads ambush professionals mid-workflow, causing embarrassing delays in front of paying clients. Basic features—like removing app logos, auto-saving original files, or browsing by project folders—are locked behind predatory paywalls. Apps suffer from intense feature creep, resulting in sluggish runtime processing and bloated installation sizes. 3. Target UsersPrimary UsersConstruction Supervisors: Track project progress, document completed work, and create client reports. Property Inspectors: Document property conditions and generate instant inspection records. Real Estate Agents: Record property visits and document listing conditions safely. Insurance Inspectors: Capture tamper-evident data and create initial claim documentation. Delivery Operators: Capture clean proof of delivery with verified location metadata. 4. MVP GoalsThe MVP focuses entirely on radical simplification: undercutting competitors by giving away core utilities for free, running 100% offline via local device engines, and keeping the local laptop development footprint lightweight and cloud-compiled. In ScopeCapture GPS-tagged photos with hardcoded overlay template styles. Auto-save both the stamped and un-stamped original photo files to local app memory. Organize photos into local project folders by default via expo-sqlite. Generate local, professional PDF and CSV reports on the device client. Work 100% offline via client-side caching and JavaScript execution libraries. Monetize via silent adaptive banner ads, a user-triggered "24-Hour Premium Ad Pass", or a low-tier "Lifetime Pro" one-time purchase. Out of ScopeVideo recording, rendering, or editing capabilities (to optimize runtime overhead). Live employee clock-in/out tracking logic (eliminates background execution battery penalties). Built-in cloud-sync providers (OneDrive/Google Drive integrations). File sharing is offloaded natively via the Android system sharesheet interface. Complex KMZ / ZIP export formats. 5. Functional Requirements5.1 Camera Viewport & ActionsFR-001: User can open camera viewport seamlessly via expo-camera. FR-002: User can capture photo. FR-003: User can switch between front and back camera lenses. FR-004: User can toggle device flash modes (On/Off/Auto). FR-005: User can zoom using multi-touch pinch gestures. FR-006: App shall auto-save the timestamped photo and a duplicate clean, un-watermarked photo to local app sandbox storage simultaneously. 5.2 GPS Location & GeocodingFR-007: App shall retrieve GPS coordinates via native hardware location providers using expo-location. FR-008: App shall retrieve and display location accuracy parameters. FR-009: App shall show timestamps accurate to the second. FR-010: When an active network connection is detected, the app shall fetch the human-readable address via reverse geocoding. FR-011: The app shall continue functioning normally offline or if the address lookup fails, using raw latitude/longitude coordinates. 5.3 On-Device Watermark Layouts (Photo Templates)FR-012: App shall burn metadata directly onto the output image file path locally using JavaScript processing libraries or canvas manipulation. FR-013: User can toggle individual overlay components on or off (Date, Time, Latitude, Longitude, Accuracy). FR-014: User can toggle between three pre-built hardcoded watermark layouts: Style 1 (Industrial Badge): A semi-transparent black rectangular overlay container anchored at the bottom-right corner. Style 2 (Clean Inspector): Overlay text rendered directly on the image background using a high-visibility font color with a crisp drop shadow stroke for readability. Style 3 (Corporate Minimalist): Clean typography aligned at the bottom edge boundary. FR-015: User can choose the visual layout anchor position (Top Left, Top Right, Bottom Left, Bottom Right). 5.4 Local Project & Photo ManagementFR-016: User can create projects containing Project Name, Client Name (optional), and Project Description (optional). FR-017: User can edit project metadata parameters. FR-018: User can delete a project (which safely cascades to purge associated photo assets locally). FR-019: User can view project photos sequentially in a clean, smart grid view. FR-020: User can filter or search through photos using text strings from user notes. FR-021: User can attach alphanumeric notes directly to individual photo records. 5.5 Report Generation & File ExportingFR-022: User can assemble and generate structured PDF and CSV reports locally on the device. FR-023: The compiled PDF file layout shall include: Project Name, Report Date, Photo grids, Timestamps, GPS Coordinates, and associated Notes. FR-024: User can safely share or export generated documents using native Android system sheets. 6. Non-Functional RequirementsPerformanceNFR-001: Camera interface initializes and displays layout within under 2 seconds. NFR-002: Photo capture, image watermark merging, and file write tasks finish in under 1.5 seconds. NFR-003: Local PDF engine compiles a 50-photo report in under 10 seconds on mid-tier test targets. Reliability & Local Storage StabilityNFR-004: State recovery logic protects against data drops during forced app lifecycle terminations. NFR-005: Database schema migration configurations prevent photo pathway disconnections during application updates. NFR-006: All application structural metadata must live locally inside an fast relational database schema via expo-sqlite. NFR-007: To comply with modern Android Scoped Storage requirements, all application photo assets are stored in the app's secure, private sandbox documents folder (expo-file-system). If the app is uninstalled, data is wiped cleanly without polluting the user's public gallery. Local System Development GuardrailsNFR-008: Zero Local Gradle Compiling: The local development computer shall not run heavy native compilation tasks. All final production APK/AAB builds shall be offloaded entirely to cloud machines using Expo Application Services (eas-cli). NFR-009: RAM Preservation: On-device preview testing shall be prioritized using the physical testing device over Wi-Fi via Expo Go, bypassing the need for heavy local Android emulators. 8.
